# Data Flow

This document describes how data enters the system, is transformed, and exits or is persisted.

## Default lifecycle

1. Data enters through a defined input boundary.
2. Validation and normalization occur first.
3. Core logic transforms or routes the data.
4. The system emits a response, artifact, or persisted state.

## Notes

Document sensitive transitions, irreversible transformations, and external handoffs explicitly.
