# Artifact Provenance

This document explains how important artifacts are created, versioned, and traced back to their source inputs.

## Track for each artifact

- Origin: where it comes from.
- Generator: which process or tool produced it.
- Inputs: what it depends on.
- Validation: how it is checked.
- Retention: how long it matters.

## Typical artifacts

- Builds
- Test reports
- Release notes
- Generated documentation
