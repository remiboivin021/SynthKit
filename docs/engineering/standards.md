# Engineering Standards

This document captures durable engineering conventions that should remain stable across contributors.

## Baseline standards

- Prefer explicitness over implicit behavior.
- Keep changes small and reviewable.
- Preserve deterministic behavior where possible.
- Document contract changes with the code change.
- Avoid adding dependencies without a clear reason.
