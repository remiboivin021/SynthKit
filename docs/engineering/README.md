# Engineering Documentation

This directory documents **how the project is engineered**, not how it is managed.

---

## Purpose

Engineering docs exist to:
- ensure consistency
- support onboarding
- reduce implicit knowledge

---

## Contents

- `standards.md`  
  Engineering standards and conventions.

- `guidelines/`  
  Domain-specific or evolving guidelines.

- `newcomer_guide.md`  
  Onboarding guide for new contributors.

- `artifact_provenance.md`  
  How artifacts are produced and tracked.

- `external_project_usage.md`  
  How this project can be reused or embedded elsewhere.

---

## Scope

This directory is recommended for team projects.
It is optional for solo or experimental work.
