# Contribution Model

This document defines who can contribute, how contributions are shaped, and what quality gates apply before acceptance.

## Contribution rules

- Changes should be scoped and reviewable.
- Public behavior changes must be documented.
- Architectural changes should link to an ADR.
- AI-generated output requires human review.

## Roles

- Author: proposes and implements scoped change.
- Reviewer: validates correctness, scope, and risk.
- Maintainer: resolves ambiguity and final acceptance.
