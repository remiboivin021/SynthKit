# Decision Process

This document explains how significant project decisions are proposed, reviewed, and recorded.

## Default flow

1. State the problem clearly.
2. Define scope and constraints.
3. Evaluate options and trade-offs.
4. Record the chosen path.
5. Link the decision to implementation and documentation.

## Escalation

Use an ADR when a decision affects architecture, boundaries, or long-term maintenance cost.
