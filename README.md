# SynthKit
An open-source kit for AI-augmented development.

## What is SynthKit?

SynthKit is a structured project template designed to help developers build software effectively with AI assistance.

It provides documentation scaffolding, architectural clarity, decision tracking, and AI usage boundaries — so humans stay in control while leveraging AI productively.

SynthKit does not generate code for you.
It helps you structure projects so AI can assist without creating chaos.

---

## Why SynthKit?

AI works better with:
- Clear context
- Explicit architecture
- Defined constraints
- Tracked decisions
- Human oversight

Most AI-driven projects fail due to missing structure, not missing intelligence.

SynthKit provides that structure.

---

## Who is it for?

- Makers
- Indie developers
- Engineers
- Tech leads
- Early-stage teams

Anyone who wants to use AI as a collaborator — not as a replacement for thinking.

---

## What’s included?

- Context scaffolding (product, technical, constraints)
- Architecture documentation template
- ADR (Architecture Decision Record) structure
- AI usage guidelines
- Agent definition framework
- Governance & security baseline

Vendor-agnostic by design.

---

## Philosophy

1. Structure over prompts  
2. Context before code  
3. Human-led, AI-assisted  
4. Decisions must be explicit  
5. Clarity scales better than magic  

---

## How to use

Click “Use this template” on GitHub.

Then:
1. Fill `PROJECT.md`
2. Define constraints in `CONTEXT/`
3. Document architecture
4. Only then start coding — with AI if needed

---

## Minimum Required Structure

You do not need to use every file.

### Required (minimum viable context)

- `PROJECT.md`
- `context/product_context.md`
- `ai/coding_rules.md`
- `docs/architecture/index.md`

These establish:
- purpose
- constraints
- coding boundaries
- system structure

### Recommended (team projects)

- `docs/governance/`
- `docs/tooling/`
- `docs/testing/`

### Optional (regulated / safety-critical domains)

- `docs/requirements/`
- `traceability_matrix.md`
- `safety_requirements.md`

SynthKit is modular.  
Use what improves clarity. Ignore the rest.
